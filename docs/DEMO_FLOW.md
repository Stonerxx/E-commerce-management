# 完整演示流程

本文用于答辩和最终联调，建议控制在 12～15 分钟。演示环境应部署 `main` 最新版本，并已在当前 Oracle Schema 中按顺序执行数据库脚本。`init_database.sql` 会重建业务表，只在允许清空演示数据时执行。

## 1. 演示前检查

先验证服务与数据库：

```text
http://服务器公网IP/
http://服务器公网IP/health/ready
http://服务器公网IP/api/v1/system/db-check
```

期望结果：

- 首页、健康检查均可访问；
- 数据库检查显示 `connected: true`；
- `sessionUser`、`currentSchema` 应为演示 Schema `ECOMMERCE_DEMO`；
- 浏览器控制台无持续报错，桌面端和窄屏下导航可正常展开；
- 页面可切换深色/浅色主题，刷新后主题仍保持。

运行发布前自动化检查：

```powershell
dotnet build ECommerce.sln -c Release
dotnet test ECommerce.sln -c Release --no-build
dotnet test tests/ECommerce.OracleIntegrationTests/ECommerce.OracleIntegrationTests.csproj -c Release
```

Oracle 测试的连接串和跳过条件见 [Oracle 集成测试使用说明](ORACLE_INTEGRATION_TESTS.md)。

## 2. 演示账号

密码统一为 `demo123`。

| 账号 | 角色 | 主要演示内容 |
| --- | --- | --- |
| `demo_user` | `USER` | 推荐、优惠券、地址、购物车、下单与支付 |
| `demo_buyer` | `USER` | 已完成订单、物流与商品评价 |
| `demo_service` | `SERVICE` | 后台订单、发货与物流轨迹 |
| `demo_admin` | `ADMIN` | 仪表盘、统计、导出、商品、库存、用户权限与审计日志 |

## 3. 游客与首页（约 1 分钟）

1. 未登录打开首页，说明页面会展示商品、热门内容和推荐入口。
2. 进入 `/products`，使用分类、关键词或排序筛选商品。
3. 打开任一商品详情，展示 SKU 选择、库存状态、评价和“为你推荐”。
4. 将浏览器缩窄到手机宽度，确认卡片、筛选区和导航自动适配。
5. 切换深色/浅色主题，说明主题设置会保存在浏览器中。

## 4. 普通用户完整下单（约 3 分钟）

使用 `demo_user / demo123` 登录。

1. 进入 `/coupons`，领取一张可用优惠券；过期或停用模板应显示“已失效”，不能领取。
2. 进入 `/addresses`，新增或选择收货地址，并点击“设为默认”。切换默认地址后，旧地址应自动取消默认，始终只有一个有效默认地址。
3. 从商品页选择 SKU 加入购物车，在 `/cart` 修改数量或勾选商品。
4. 结算进入确认订单页，核对默认地址、商品快照、优惠券和服务端计算的实付金额。
5. 提交订单后进入 `/payment/{orderId}`，点击“模拟支付成功”。
6. 返回 `/orders`，确认新订单状态为“已支付”。

说明：模拟支付仍会真实写入 `PAYMENT`，订单状态、优惠券核销和库存扣减由事务保护，不是只改页面文字。

## 5. 订单、物流与评价（约 2 分钟）

使用 `demo_buyer / demo123` 登录。

1. 打开 `/orders`，按状态查看已完成和已取消订单。
2. 进入已完成订单详情，展示收货信息快照、商品明细、状态日志和物流轨迹。
3. 对已完成且尚未评价的商品提交评分和文字评价。
4. 返回商品详情，确认审核通过的评价可展示；匿名评价不暴露用户名。

## 6. 客服后台（约 2 分钟）

使用 `demo_service / demo123` 登录。

1. 打开 `/admin/dashboard`，说明客服看到的是业务处理首页，而不是普通用户商城首页。
2. 进入 `/admin/orders`，使用订单号、用户、状态或时间条件筛选。
3. 打开一张已支付订单，执行发货并填写物流公司、运单号。
4. 新增物流轨迹，切换到对应用户订单详情确认轨迹可见。
5. 验证客服可以处理订单，但不能访问仅管理员可见的统计、用户权限和商品维护页面。

## 7. 管理员后台（约 4 分钟）

使用 `demo_admin / demo123` 登录。

1. 打开 `/admin/dashboard`，展示订单、销售额、库存预警和待办摘要；说明该页是运营总览。
2. 打开 `/admin/statistics`，分别切换“按日”和“按月”统计，展示趋势和热销商品排行。
3. 点击“导出订单明细”和“导出库存变动”，确认浏览器下载有效 Excel；页面导出上限为 5,000 行。
4. 打开 `/admin/products`、`/admin/skus`、`/admin/inventory/warnings` 和 `/admin/inventory/logs`，展示商品上下架、SKU、库存预警、人工调整与变动追踪。
5. 打开 `/admin/coupons` 和 `/admin/reviews`，展示优惠券模板维护与评价审核。
6. 打开 `/admin/users` 和 `/admin/permissions`，展示账号状态、角色和权限配置。
7. 打开 `/admin/operation-logs`，筛选刚才的后台写操作，确认操作人、模块、动作、时间和请求信息已记录。

统计页与后台首页的定位不同：后台首页用于快速查看关键指标和待办，统计页用于选择日/月维度、查看趋势排行和导出明细，不应展示成两个相同页面。

## 8. 权限与异常检查（约 1 分钟）

1. 退出登录后访问 `/orders`，应跳转登录页。
2. 用普通 `USER` 访问 `/admin`，应进入拒绝访问页面。
3. 重复领取同一张优惠券、使用过期优惠券或超库存下单时，页面应给出明确提示，不产生半成功数据。
4. 同一用户连续设置两个默认地址后，在数据库和页面上仍只能看到一个有效默认地址。

## 9. 演示收尾

答辩结束前给出三项证据：

- GitHub Actions 的 `main` 发布任务成功；
- `/health/ready` 与 `/api/v1/system/db-check` 正常；
- 单元测试和已启用的 Oracle 集成测试失败数为 0。

若某个 Oracle 用例显示 `SKIP`，应说明对应环境变量或长耗时开关未启用，不能把“跳过”描述成“已验证”。
